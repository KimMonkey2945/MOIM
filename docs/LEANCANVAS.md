## 1. 고객군 (Customer Segments)

**타깃 고객**

- 범용 소모임의 **주최자(호스트)** — 운동 동호회·친목 모임·스터디 그룹을 정기적으로 운영하는 사람
- 그 모임의 **참여자** — 회원가입 후 RSVP·카풀·정산에 참여하는 사용자

**얼리어답터 (Early Adopters)**

- **월 2회 이상 정기적으로 모이는 운동 동호회 주최자** (수영·헬스·러닝, 10~15명 규모)
- 매 모임마다 **카풀과 정산이 반복 발생**해 운영 부담이 큰 모임
- 페르소나 A "김도현"(주말 수영 소모임 12명, 매월 2회)이 대표 프로파일

> 운동 동호회는 ① 일정·장소가 반복되고 ② 카풀·정산 빈도가 높아 통합 도구의 가치가 가장 즉시 체감되는 진입점이다.

## 2. 문제 (Problem)

소모임 주최자는 하나의 이벤트를 운영하려고 여러 채널을 동시에 오간다.

**Top 3 문제**

1. **정보 분산** — 공지·참여자·카풀·비용이 카톡·폼·엑셀·정산앱 3~4개에 흩어져 매몰·유실된다.
2. **주최자 번아웃** — 참석 취합 → 카풀 조율 → 정산까지 매 모임 **2~3시간**의 반복 수작업.
3. **참여자 깜깜이** — "내가 참석 신청을 했나" "카풀 배정됐나" "얼마 내야 하나"를 주최자에게 매번 되묻는다.

**기존 대안재 (Existing Alternatives)**

| 업무      | 현재 사용 도구          | 한계                          |
| --------- | ----------------------- | ----------------------------- |
| 공지·일정 | 카카오톡 단톡방         | 메시지 매몰, 링크 유실        |
| 참여 집계 | 네이버 폼 / 댓글        | 수동 집계, 실시간 파악 불가   |
| 카풀      | 단톡방 + 엑셀           | 좌석 중복, 연락 반복          |
| 정산      | 더치트리 / 스프레드시트 | 입력 이중화, 송금 확인 수작업 |

> 핵심: 어느 대안도 **4개 영역을 한 곳에서** 묶지 못해, 주최자가 직접 채널을 옮겨 다니며 손으로 연결한다.

## 3. 고유 가치 제안 (Unique Value Proposition)

> **"공지·참여·카풀·정산을 한 화면에서. 주최자의 운영 부담을 없애는 원스톱 모임 관리 도구."**

- 단일 기능 도구(정산만, 폼만)가 아니라 **모임 운영 전체 워크플로**를 하나로 통합.
- 차별화 히어로 = **주최자 대시보드**: 참여 현황·미승인 카풀·미정산을 한 화면에서 즉시 파악.

**하이레벨 컨셉 (X for Y)**

- "**소모임 운영을 위한 노션 + 더치트리**" — 흩어진 운영 업무를 하나의 이벤트 허브로.
- 다른 표현: "모임 주최자를 위한 올인원 운영 비서".

## 4. 솔루션 (Solution)

각 문제에 대응하는 솔루션:

| 문제          | 솔루션 (PRD 기능)                                                                                                                             |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| 정보 분산     | **이벤트별 통합 허브** — 상세 페이지에 공지/카풀/정산 탭 (E-04)                                                                               |
| 주최자 번아웃 | **RSVP 자동 집계**(E-05·E-06), **카풀 신청→승인 플로우**(C-01~C-05), **정산 자동 분배**(균등/차등, X-02·X-03), **주최자 대시보드**(D-01~D-04) |
| 참여자 깜깜이 | **참여자별 내 상태** — RSVP·카풀 배정·내 정산 금액을 한눈에(E-05, C-04, X-03)                                                                 |

핵심 동선: 이벤트 생성(5분 내) → 참여자 RSVP 원클릭 → 카풀 좌석 신청·승인 → 비용 자동 분배·송금 체크 → 대시보드에서 미완료 항목 추적.

## 5. 채널 (Channels)

**도달 경로**

- **모임 커뮤니티**: 네이버 카페, 당근 모임, 소모임/문토 앱, 운동 오픈채팅·디스코드
- **주최자 공유 바이럴**: 주최자가 이벤트를 운영하면 참여자가 자연스럽게 제품에 노출 → 새 주최자로 전환 (회원가입 필수 구조)
- **오프라인 제휴**: 수영장·헬스장·스터디카페 등 모임이 열리는 시설 제휴
- **콘텐츠/SEO**: "동호회 총무 정산·카풀 관리 팁" 등 운영 노하우 콘텐츠
- **간편 진입**: 기존 Google OAuth 로그인으로 가입 마찰 최소화

> ⚠️ v1은 **게스트 초대 링크가 비목표**라 참여자 → 주최자 바이럴 루프가 약하다. 초기엔 커뮤니티 직접 공략에 집중하고, 게스트 링크(v2)로 바이럴을 강화한다.

## 6. 수익원 (Revenue Streams)

**v1: 무료** — 멤버십·광고는 PRD 비목표. 먼저 **주최자 재방문율(리텐션)**로 가치를 검증한다.

**향후 수익화 후보 (검증 후)**

- **주최자 프리미엄 구독**: 다수 모임 관리, 고급 정산 리포트·통계, 알림(이메일/푸시), 반복 일정
- **정산 송금 수수료**: 토스/카카오페이 송금 연동 시 거래액 일부
- **B2B 단체 플랜**: 사내 동아리·대학 동아리·동호회 연합 단위 유료 플랜
- **제휴 광고**: 운동 시설·회식 장소·여행 등 모임 연관 제휴

## 7. 비용 구조 (Cost Structure)

**고정비 (초기 최소)**

- Vercel 호스팅, Supabase(DB·Auth·Storage) — 초기 무료~저가 티어
- 도메인, 솔로 개발자 시간(기회비용)

**변동비 (성장 시)**

- Supabase 사용량: 월간 활성 사용자(MAU), DB row/스토리지, 이미지(썸네일·아바타) 대역폭
- 이메일/푸시 발송 비용(알림 도입 시), 결제·송금 연동 수수료(수익화 시)
- 커뮤니티 마케팅·콘텐츠 제작 비용

> MVP 손익 구조상 사용자당 한계비용이 매우 낮아, 리텐션만 확보되면 무료 운영 후 점진 수익화가 가능하다.

## 8. 핵심 지표 (Key Metrics)

PRD 11장 KPI를 린 캔버스 지표로 매핑:

| 지표                       | 목표                          | 단계(AARRR)   |
| -------------------------- | ----------------------------- | ------------- |
| 이벤트 생성 완료율         | 시작 대비 80%+                | Activation    |
| RSVP 응답률                | 참여자 70%+                   | Engagement    |
| 카풀 처리 완료율           | 신청 90%+ 승인/거절           | Engagement    |
| 정산 완료율                | 종료 후 7일 내 80%            | Engagement    |
| **주최자 재방문율 (OMTM)** | **1개월 내 2번째 이벤트 50%** | **Retention** |

> **가장 중요한 단 하나의 지표(OMTM): 주최자 재방문율.** 주최자가 두 번째 모임을 다시 이 도구로 연다는 것은 "운영 부담 경감" 가치가 실제로 전달됐다는 가장 강한 신호다.

## 9. 차별적 경쟁우위 (Unfair Advantage)

> 솔직히, 초기 MVP 단계에서 복제 불가능한 해자는 약하다. 아래는 **현재 강점 + 시간이 쌓이며 형성될 해자 후보**다.

**현재 강점**

- **통합 워크플로의 깊이**: 더치트리(정산만)·네이버폼(집계만) 등 단일 기능 도구가 4개 영역을 한 워크플로로 묶기는 구조적으로 어렵다.
- **주최자 부담 경감에 집중한 UX**: 범용 협업툴(노션 등)이 흉내 내기 어려운 모임-특화 동선·대시보드.

**시간이 만드는 해자 후보 (방어선 구축 대상)**

- **누적 데이터**: 모임 이력·참여 신뢰도·정산 기록이 쌓일수록 전환 비용↑
- **네트워크 효과**: 한 사용자가 여러 모임에 참여 → 계정·관계 그래프가 락인
- **주최자 습관화**: 반복 모임을 운영하는 루틴 도구로 자리잡으면 교체 저항↑
